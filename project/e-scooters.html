E scooters

Mobile app as a school project

Posted on

As a school project, I was part of a team tasked with coming up with an idea for a mobile application. We decided to create an application that regroups all the e-scooter brands in Stockholm. The idea was that the user would only need to have a single app to compare the location and prices of the e-scooters nearby and access it. First, we created a web app. Using React and Tailwind CSS libraries, we designed the front-end. I focused on using the Leaflet library to display the map, the user location, and e-scooter pins on their locations, while other members of the team built other parts of the app, such as the communication with e-scooter brands' API. In the end, we ended up with a standalone web app that displayed e-scooters' location, but without the functionalities of rental. Secondly, we had to create a "native app". We chose to use React Native to be cross-platform. Once again, I was in charge of taking care of the map utilities, but this time, we didn't find easy-to-use and cross-platform libraries to display a map like Leaflet does. Thus, I implemented the Map tiles workflow, using Open Street Map data, in order to reproduce the behavior of a map component. It worked well, except for zooming utilities. We reached almost the same point with the native app as with the web app. This experience allowed me to work as part of a team and to develop my skills in web frontend and cross-platform development. I also gained experience in using React and React Native, as well as in using libraries like Leaflet and Open Street Map for maps. It also helped me to understand the challenges and differences in developing a web app and a native app.
