Skillect

Full stack portfolio website

Posted on

As a personal project, I aimed to create a backend with a relational database to store and serve my experiences and skills. The goal was to use this backend in a portfolio website, both displaying my skills to anybody and allowing myself to edit the data. From a full-stack template using FastAPI, Postgres, and Vue.js, I developed a full-stack website. The backend is based on FastAPI, which serves a REST API on top of a WSGI Http server (for which I used Gunicorn). Using the SQLAlchemy library, I designed the data model, both for user authentication and authorization workflows and to store professional skills. Good practices of using linters/formaters (Black, Pylint and Isort here), continuous integration (with GitHub Actions) and containerization (with Docker) were used here. The front-end is based on React, MUI and Tailwind CSS libraries (in opposition to Vue.js that was used in the original template). It uses Axios to communicate with the backend, including for authentication. Here too, good practices of using linters/formaters (Prettier, ESLint), continuous integration and containerization were applied. I also set up a reverse proxy, based on Traefik, in charge of redirecting API calls properly to the backend. It also uses Let's Encrypt to automatically renew a SSL certificate (needed to enable HTTPS protocol). The environment is described using Docker Compose, that includes the backend, the front-end, the reverse proxy, and other services like the Postgres database. During this project, which was ultimately abandoned, I learned how to build a real-world full-stack website, from the technical functionalities (e.g. authentication) to the deployment (GitHub Actions, AWS, domain name, SSL certificates...). I also learned how to set up a reverse proxy and how to work with various libraries and technologies like FastAPI, React, and SQLAlchemy.
